# Bug 单

## BUG-001 用户名长度小于6位时注册失败

- 关联用例：TC-002
- 所属模块：用户注册
- 严重级别：Major
- 优先级：P1
- 请求接口：/api/register
- 请求数据：`{'username': 'u0559', 'password': '123456'}`
- 预期结果：状态码 400，响应包含 `用户名长度必须为6-20位`
- 实际结果：状态码 200，响应内容 `{'message': '注册成功'}`

### 修复建议

根据关联需求补充接口校验或修正业务逻辑，确保接口返回状态码 400，且响应内容包含 `用户名长度必须为6-20位`。
