# 登录注册 Demo 系统 PRD

## 1. 项目背景

本系统用于验证 AI 自动化测试全流程插件原型。系统提供用户注册、登录和登录后首页访问功能，用于验证从 PRD 解析、需求拆解、测试用例生成、自动化执行、测试报告生成到 Bug 单生成的完整流程。

## 2. 产品目标

1. 用户可以注册账号。
2. 用户可以使用已注册账号登录系统。
3. 登录成功后可以进入首页。
4. 系统需要对异常输入进行校验。
5. 系统预埋一个业务规则类 Bug，用于验证自动化测试发现缺陷的能力。

## 3. 用户注册需求

1. 用户名不能为空。
2. 用户名长度必须为 6-20 位。
3. 用户名不能重复。
4. 密码不能为空。
5. 密码长度不能少于 6 位。
6. 注册成功后提示“注册成功”。

## 4. 用户登录需求

1. 用户名不能为空。
2. 密码不能为空。
3. 已注册用户输入正确密码时，登录成功。
4. 未注册用户登录失败。
5. 密码错误时登录失败。
6. 登录成功后进入首页。

## 5. 预埋缺陷说明

需求要求用户名长度必须为 6-20 位，但当前后端注册接口故意没有校验用户名小于 6 位的情况。

预期结果：用户名小于 6 位时注册失败。

实际结果：用户名小于 6 位时可以注册成功。
