# 登录注册 Demo 系统需求分析

## 1. 需求清单

| 需求编号 | 模块 | 需求描述 | 优先级 | 是否可测试 |
|---|---|---|---|---|
| REQ-001 | 注册 | 用户名不能为空 | P1 | 是 |
| REQ-002 | 注册 | 用户名长度必须为 6-20 位 | P1 | 是 |
| REQ-003 | 注册 | 用户名不能重复 | P1 | 是 |
| REQ-004 | 注册 | 密码不能为空 | P1 | 是 |
| REQ-005 | 注册 | 密码长度不能少于 6 位 | P1 | 是 |
| REQ-006 | 注册 | 注册成功后提示“注册成功” | P2 | 是 |
| REQ-007 | 登录 | 用户名不能为空 | P1 | 是 |
| REQ-008 | 登录 | 密码不能为空 | P1 | 是 |
| REQ-009 | 登录 | 正确账号密码可以登录成功 | P1 | 是 |
| REQ-010 | 登录 | 未注册用户登录失败 | P1 | 是 |
| REQ-011 | 登录 | 密码错误时登录失败 | P1 | 是 |
| REQ-012 | 首页 | 登录成功后进入首页 | P2 | 是 |

## 2. 重点测试风险

REQ-002 是本项目的重点验证需求：用户名长度必须为 6-20 位。

当前系统预埋缺陷：后端未校验用户名小于 6 位的情况。
